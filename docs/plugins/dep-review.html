dep-review

Evaluate Dependabot PRs with AUTO-MERGE / MERGE / SKIP / INVESTIGATE verdicts grounded in the diff, changelog, and codebase usage.

Install

/plugin install dep-review@nautilai

What it does

Lists open Dependabot PRs, classifies each bump (patch/minor/major), checks CI status, extracts changes from the changelog, and verifies each against your code (cited file:line). Evaluates security (CVE and whether the vulnerable path is used) and EOL status. Applies a decision matrix to land AUTO-MERGE / MERGE / SKIP / INVESTIGATE, handles cross-PR concerns like react/react-dom parity, and produces a report. Low-risk patch and minor-dev-dep bumps with passing CI auto-merge; other actions gate behind approval.

How it works

The skill discovers open Dependabot PRs via GitHub MCP (preferred) or gh CLI, parsing package, versions, and ecosystem from branch prefix. For each PR, it classifies the bump type, checks CI status, extracts the specific changes from the changelog, and verifies each against your code (cited file:line). It evaluates security (CVE + whether the vulnerable path is used) and EOL status.

A decision matrix applies the verdict: AUTO-MERGE for low-risk patch and minor-dev-dep bumps with passing CI; MERGE, SKIP, or INVESTIGATE for everything else. The skill reports findings grounded in diff and changelog; when metadata is unreachable it degrades loudly rather than guessing.

Each PR's per-PR analysis runs as its own parallel subagent on model: sonnet, launched together in batch mode; the main context keeps discovery, verdicts, and gated actions.

Shoals: records corrections in .claude/shoals/dep-review.dep-review.md for the next run, learning what to always skip or adjust verdict-wise on specific packages.

Under the hood
  • PR discovery via GitHub MCP (preferred) or gh CLI; parses ecosystem from branch prefix.
  • Per-PR analysis runs as a parallel general-purpose subagent (model: sonnet) per PR; batch mode launches all subagents together.
  • Bump classification: patch, minor (runtime), minor (dev-dep), major.
  • Changelog extraction from diff; registry metadata via npm / pip-audit (if present) or HTTP APIs (npm, PyPI).
  • Verification: each change cited against codebase (file:line) or marked "not found".
  • Security: CVE checks + tracks whether vulnerable path is used; EOL status from registry.
  • Evidence-based verdicts: cite specific changes from diff/changelog, backed with file:line usage claims; unverifiable metadata is reported, not guessed.
  • Bounded auto-merge: only patch/minor-dev-dep bumps with passing CI auto-merge, narrowing the high-risk class; CI passing is the consent.
  • AUTO-MERGE is auto-fix (exception to opt-in mutation); other merges/closes are ask-user approval gates.
  • Shoals file: .claude/shoals/dep-review.dep-review.md — learns package-specific corrections.

Requirements

Authenticated GitHub client: mcp__github__* MCP server (preferred) or gh CLI. Optional: npm or pip-audit for enhanced registry metadata (falls back to HTTP APIs if absent).

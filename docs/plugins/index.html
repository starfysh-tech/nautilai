Plugins

All 15 plugins in the marketplace — pick one for the install command and full details.

  • commitcraftAI git workflow toolkit: conventional commits, issue validation, PR creation, and release guidance.
  • relaySession continuity for Claude Code: transcript-grounded handoff documents that a fresh session picks up automatically. Ships the handoff skill.
  • cc-adoption-auditAudit your Claude Code setup against available features and surface what you're not using but should, plus setup gaps.
  • pr-comment-reviewProcess and address review comments on the current pull request — fetch the threads, categorize, implement agreed fixes behind approval gates, push, and reply inline.
  • cc-validate-hooksValidate the local Claude Code hooks configuration in settings.json — report schema errors, invalid event names, malformed matchers, and bad hook fields, with an optional --fix.
  • cc-skill-auditAudit existing Claude Code skills against Anthropic's authoring guidance — diagnose under/over-triggering, tighten descriptions, de-bloat bodies, and sweep a skills directory (including installed plugins) for issues.
  • phi-scanScan a repo for Protected Health Information (PHI under HIPAA Safe Harbor) — SSNs, emails, phones, IPs, dates, restricted ZIPs — then AI-triage findings to filter false positives. Optionally runs Django/React OWASP grep checks when that stack is detected.
  • review-planAdversarially validate an implementation plan against the actual codebase before building it — surface risks, breaking changes, gaps, and simplifications, then write findings back into the plan. Built-in agents by default; taps review/Codex plugins when installed.
  • pr-review-deepRigorous, evidence-based code quality review of a branch or PR — abstraction design, type/boundary contracts, and behavior-preserving structural simplification. Proposes high-leverage restructurings with cited evidence; never performs them or expands scope.
  • rbac-djangoAudit a Django/DRF + React codebase for RBAC gaps (missing permission classes, tenant-isolation leaks, IDOR, write-side gaps), threat-model the findings into ranked attacker abuse cases, then generate prioritized remediation playbooks and GitHub issues.
  • frontend-reviewAudit a React + Tailwind frontend for architecture and design-token health — component size, prop drilling, duplicate-primitive reuse, folder organization, and Tailwind token violations. Auto-detects the frontend source root; findings cite file:line.
  • wireframeCreate low-fidelity wireframes for UI planning — ASCII layouts, wiremd interactive prototypes, or Mermaid diagrams for tickets. Use before writing components to sketch structure, flows, and states.
  • dep-reviewEvaluate Dependabot PRs with AUTO-MERGE / MERGE / SKIP / INVESTIGATE verdicts grounded in the PR diff and codebase usage. Low-risk patch / minor-dev-dep bumps that pass CI auto-merge; other merges and closes gate behind approval.
  • github-issue-auditorAudit a GitHub repo's issues for cleanup — fuzzy-matched duplicates, orphaned sub-issues, unlabeled or stale items, and inconsistent labels against the repo's own auto-detected taxonomy. Read-only by default; any mutation is opt-in behind an approval gate.
  • autodevBounded autonomous development loop: scripted worktree lanes, fast worker subagents, objective script-based verification, and a hard 3-failure escalation to the user.

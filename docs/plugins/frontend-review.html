frontend-review

Audit React components and Tailwind tokens for architecture and design-token health.

Install

/plugin install frontend-review@nautilai

What it does

Two read-only audits for a React + Tailwind frontend analyze component architecture and design-token health. The react-component-architecture skill flags oversized components, prop drilling, and duplicated UI patterns that should be extracted to primitives; the tailwind-design-token-validator detects arbitrary values, dynamic class concatenation, invalid @apply use, and hardcoded colors. Both auto-detect the frontend source root and Tailwind config without assuming hardcoded paths, cite every finding with file:line, and run on Glob/Grep/Read with no external dependencies.

Commands / skills

  • /react-component-architecture

    Audit component architecture for oversized components, prop drilling, duplicated UI patterns, and folder organization issues.

    explicit slash form only
  • /tailwind-design-token-validator

    Audit Tailwind usage for arbitrary values, dynamic class concatenation, @apply overuse, inline styles, and design-token violations.

    explicit slash form only

How it works

  • Each skill resolves the frontend source root by locating the React package.json rather than assuming a hardcoded path like client/ or src/.
  • The Tailwind skill also discovers the project's tailwind.config.* file (or v4 CSS @theme) and treats it as the source of truth for what constitutes a valid token.
  • Both skills scan .tsx, .jsx, and .css files, reporting findings-first — every claim about the codebase cites a specific file:line.
  • The Tailwind skill can auto-fix exact token maps (e.g. bg-[#3b82f6] → bg-primary when the values are identical and a token exists), applied only after user approval and visible in version control.

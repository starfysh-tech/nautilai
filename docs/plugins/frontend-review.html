frontend-review

Audit React components and Tailwind tokens for architecture and design-token health.

Install

/plugin install frontend-review@nautilai

What it does

Two read-only audits for a React + Tailwind frontend analyze component architecture and design-token health. The react-component-architecture skill flags oversized components, prop drilling, and duplicated UI patterns that should be extracted to primitives; the tailwind-design-token-validator detects arbitrary values, dynamic class concatenation, invalid @apply use, and hardcoded colors. Both auto-detect the frontend source root and Tailwind config without assuming hardcoded paths, cite every finding with file:line, and run on Glob/Grep/Read with no external dependencies.

Commands / skills

  • /tailwind-design-token-validator

How it works

Both audits auto-detect the frontend root and configuration, then scan and report findings with file:line citations.

Under the hood
  • /react-component-architecture — detects oversized components, prop drilling, duplicated UI patterns, folder organization issues; runs on Glob/Grep/Read (disposition: report).
  • /tailwind-design-token-validator — detects arbitrary values, dynamic class concatenation, invalid @apply use, inline styles, design-token violations; runs on Glob/Grep/Read with optional Edit for auto-fixed token swaps (dispositions: auto-fix for 1:1 token maps, report for findings, ask-user for approximate changes).
  • Auto-detection: locates React package.json for source root; discovers tailwind.config.* or v4 CSS @theme as source of truth for tokens.
  • Shoals: .claude/shoals/frontend-review.<skill>.md — project corrections (thresholds, token mappings, patterns to ignore).
Design notes
  • Ported from Python modules (importable libraries, no CLI) to model-driven logic via Glob/Grep/Read for better composability and no hardcoded paths.
  • Both skills are user-invoked only (not auto-firing) and read-heavy, using per-project shoals to carry forward team corrections across runs.
